# macOS Focus Monitor

**⌨️ 你的 Mac 输入法是不是总突然丢失焦点，打字没响应？这个工具帮你抓到"幕后黑手"**

---

## 这是什么？我为什么要用它？

你有没有遇到过这种情况：

> 正在专心打字，突然窗口焦点跳走了，刚打的内容跑到了别的框里……
> 或者某个 App 自己弹到前台，把你正在看的网页/文档给挡住了……

这不是你手滑，是有些 App 在后台偷偷"抢焦点"（专业说法叫 Focus Stealing）。

**更可怕的是**，有些 App 还会在你不知情的时候自动截图——你输入的密码、聊天的内容、浏览的网页都可能被截走。

这个工具就是帮你 **24 小时盯梢**，一旦发现哪个 App 在背后搞小动作，立刻记下来，最后给你一份"黑名单报告"。

---

## 🚀 5 分钟上手（完全不用懂编程）

> 别怕终端命令，你只需要复制粘贴，每一步都解释清楚了。

### 第一步：下载工具

打开 Mac 自带的 **「终端」** App（在 启动台 → 其他 → 终端，或者直接 Spotlight 搜索"终端"），复制粘贴下面这行，按回车：

```bash
git clone https://github.com/Rain-at-Spring-Dawn/macos-focus-monitor.git && cd macos-focus-monitor

> 💡 如果提示 "already exists"，说明你之前下载过了，直接运行：
>    `cd macos-focus-monitor` 然后继续下一步即可。
```

📌 *这行命令的意思是：把本工具下载到你电脑上，然后进到工具文件夹里。*

### 第二步：安装依赖

继续在终端里复制粘贴这行，按回车：

```bash
pip3 install pyobjc-framework-Quartz pyobjc-framework-Cocoa
```

📌 *这行命令的意思是：给工具装上它需要的一些"零件"。安装过程会下载一些文件，等它跑完就行。*

### 第三步：给权限（必须做！不然无法运行）

1. 点击屏幕左上角的 🍎 苹果图标 → **系统设置**
2. 点左侧 **隐私与安全性**
3. 点 **辅助功能**
4. 点击下方的 🔒 锁头图标，输入你的电脑密码
5. 点击 **+** 号，找到并添加 **「终端」**（一般在「应用程序 → 实用工具」里）
6. ✅ **勾选上终端** 前面的复选框

看不懂可以看这个路径：`系统设置 → 隐私与安全性 → 辅助功能 → 解锁 → 添加终端 → 勾选`

### 第四步：运行！

在终端里粘贴这行，按回车：

```bash
python3 focus_monitor.py -v
```

你会看到屏幕开始刷刷刷地显示信息，这就代表工具在正常工作了。

### 第五步：看结果

让它跑一会儿（比如半小时或一天），然后在终端里按 **`Ctrl + C`**（两个键一起按）停止它。

工具会自动打印一份 **「黑名单报告」**，里面列了哪些 App 在偷焦点、偷了多少次、它们的"作案手段"是什么。

## 📖 进阶玩法（可选）

如果你觉得默认设置太敏感或不够灵敏，可以加一些参数：

```bash
# 只显示可疑的（不显示正常的焦点切换）
python3 focus_monitor.py

# 调低灵敏度（减少误报，适合挂机）
python3 focus_monitor.py -t 1.0

# 同时把日志保存到文件，方便以后翻看
python3 focus_monitor.py -v -o 监控记录.txt
```

| 参数 | 有什么用？ | 默认值 |
|------|-----------|:------:|
| `-v` | 显示所有切换记录（正常的和可疑的都显示） | 关闭 |
| `-t 0.5` | 调大 → 减少误报；调小 → 更敏感 | 0.3 秒 |
| `-o 文件名.txt` | 把输出同时存成一个文件，可以慢慢看 | 不存 |
| `-i 0.2` | 每隔多久检查一次（数字越小检查越频繁） | 0.1 秒 |

## 📊 报告怎么看？

报告长这个样子，我逐行解释一下：

```
⚠  发现 2 个可疑应用:

  📱 某某软件
     可疑/总计: 48/52  (92%)      ← 它激活了52次，其中48次不是你点的
     手段: LSUIElement(没有图标)   ← 它偷偷摸摸在后台运行，没有Dock图标
     路径: /Applications/某某软件.app
     🔍 定位: open -R "路径"       ← 复制这行到终端执行，就能在访达里找到它
     🚫 禁用: mv "路径" "路径_disabled"  ← 执行这行就能禁用掉它
```

### 可疑程度判断

| 比例 | 什么意思？ |
|:----:|-----------|
| 🔴 **> 70%** | 基本上就是它在捣乱，可以考虑禁用 |
| 🟡 **30%-70%** | 有点可疑，建议继续观察 |
| 🟢 **< 30%** | 大概率是正常行为，不用管 |

### 报告里的标记是啥意思？

| 标记 | 大白话解释 | 危险程度 |
|------|-----------|:-------:|
| `LSUIElement` | 这个 App 没有 Dock 图标，躲在后台偷偷运行 | 🔴 高 |
| `FloatPanel` | 它的窗口是浮动在最上层的，容易挡住你的操作 | 🔴 高 |
| `Idle 10s` | 你 10 秒没动鼠标键盘，它自己蹦出来了 | 🟡 中 |
| `Delay 0.5s` | 你操作之后半秒，它自己激活了 | 🟡 中 |

### 报告里给的命令怎么用？

报告会直接给出两条命令：

- **`open -R "路径"`** — 复制粘贴到终端执行，会在访达里定位到那个 App
- **`mv "路径" "xxx_disabled.app"`** — 给 App 改个名，它就没法再被自动触发了（想恢复就把 `_disabled` 去掉）

---

## 🛠 它是怎么工作的？（简单版）

```
每 0.1 秒检查一次：
  1. 现在哪个 App 在最前面？
  2. 如果切换了：
     → 检查你刚才有没有动鼠标键盘
     → 如果好久没动，App 自己蹦出来了 → 可疑！
     → 记录它的各种特征（有没有 Dock 图标？窗口是不是浮动的？）
```
